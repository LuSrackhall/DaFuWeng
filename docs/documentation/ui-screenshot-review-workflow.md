# UI 截图复核工作流

## 目的

这套工作流用于为重要的前端与交互变更补上一层稳定、可复核的人类可用性证据。

它的目标不是替代单元测试、集成测试或业务 E2E，而是回答另一个问题：

- 玩家能不能在几秒内看懂当前发生了什么
- 玩家能不能迅速找到下一步最该点的地方
- 桌面端与移动端是否都没有明显遮挡、溢出、错误焦点或信息拥挤

## 何时使用

以下类型的改动默认应触发这套流程：

- 房间主界面、棋盘周边、阶段卡、主要弹层、关键 CTA 的可见变化
- 影响“轮到谁行动、当前发生了什么、下一步做什么”的信息层级变化
- 影响移动端布局、可点击性、可读性、滚动路径或安全区表现的改动
- 影响大厅入口、房间恢复、只读观战、重连反馈的可见变化

以下改动通常不需要默认触发：

- 纯后端、协议、日志、埋点或不可见的测试重构
- 不触及玩家可见表面的纯类型或内部模块调整

## 默认截图矩阵

首期默认矩阵保持小而稳定：

- 桌面端玩家常规回合界面
- 桌面端玩家高密度决策界面
- 移动端只读或观战中的房间界面

后续功能若触达拍卖、交易、欠款恢复、监狱决策等更高压力状态，应在对应切片中追加定向截图场景。

## 执行方式

在 `frontend/` 中运行：

```bash
pnpm test:e2e:ui-review
```

截图会输出到：

```text
frontend/test-results/ui-review/
```

## 复核问题清单

每组截图至少回答以下问题：

1. 玩家第一眼会先看哪里。
2. 玩家能否在 3 秒内知道当前轮到谁、发生了什么、下一步点哪里。
3. 主要 CTA 是否足够突出。
4. 是否有关键内容被遮挡、截断、挤压或需要多次扫描才能理解。
5. 移动端是否存在横向溢出、点击成本过高、信息顺序错误的问题。
6. 是否存在错误焦点，也就是不重要内容比重要内容更醒目。

## 严重程度建议

- 阻断级：玩家无法判断当前状态、无法找到主操作、关键内容被遮挡或无法阅读。
- 严重级：信息层级明显混乱、多个焦点抢戏、棋盘或决策面可读性显著下降。
- 中等级：局部拥挤、局部可点击性不明确、存在中等理解成本。
- 轻微级：主要流程可完成，但存在视觉噪音、对齐或密度上的可优化点。

## 与其他测试层的关系

- 单元测试与集成测试仍是确定性规则和状态逻辑的主回归层。
- 业务 E2E 仍负责证明关键玩家旅程与恢复路径能够走通。
- UI 截图复核负责提供“人类是否看得懂、点得准、不会被误导”的视觉证据。

这三层互补，不能互相替代。
