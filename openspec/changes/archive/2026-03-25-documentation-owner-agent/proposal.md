## Why

The repository now spans README onboarding, architecture docs, OpenSpec workflows, release artifacts, and planned public documentation.

Without a dedicated documentation owner role, README, long-lived docs, and outward-facing documentation can drift away from shipped facts and release outputs.

## What Changes

- Add a dedicated Documentation Owner agent for README and official documentation governance.
- Define the role boundary relative to Product, Tech Lead, QA, and Release Marketer.
- Document a recommended first-pass docs-site stack and minimal information architecture for future implementation.

## Capabilities

### New Capabilities
- `documentation-governance`: the repository gains a dedicated agent for README and official documentation ownership.

## Impact

- Affected code: agent customization files and OpenSpec artifacts only.
- Runtime behavior: none.
- Workflow behavior: clearer ownership for README and official docs planning.
